## Охват тестирования

Чек-лист покрывает:
- Переходы между комнатами (room transitions)
- Анимации спавна агентов (agent spawn/despawn)
- Анимации делегирования задач (task delegation / delivery arc)
- API-эндпоинты состояния анимаций комнат
- Корректность языкового контента (отсутствие корейских символов)
- Оба режима: **Режим A — «Игровой»** и **Режим B — «Корпоративный»**

## 1. Подготовка к тестированию

- [ ] Приложение запущено локально (`npm run dev` или `docker-compose up`)
- [ ] Офисный вид открыт (вкладка Office)
- [ ] В системе есть хотя бы 2 активных агента в разных отделах
- [ ] Настроена хотя бы 1 задача для проверки делегирования
- [ ] Инструменты разработчика браузера открыты для наблюдения за ошибками JS

## 2. Переходы между комнатами

### 2.1 Режим A — «Игровой и яркий»

- [ ] Клик по комнате отдела → правая панель открывается с translateX-анимацией (~300ms)
- [ ] Повторный клик по той же комнате → панель закрывается без артефактов
- [ ] Переключение темы light → dark: canvas плавно fade out → fade in (~300ms × 2)
- [ ] После смены темы цвета всех комнат обновились (не застряло старое состояние)
- [ ] При клике по комнате: кратковременный glow (border accent, alpha 0.6→0, ~30 кадров)

### 2.2 Режим B — «Корпоративный»

- [ ] Открытие панели комнаты — анимация сдержанная (~250ms), без bounce
- [ ] Смена темы: canvas fade ускорен (~200ms × 2)
- [ ] Нет ярких вспышек при room-enter (room-enter flash отсутствует)

---

## 3. Анимации спавна агентов

### 3.1 Режим A — «Игровой и яркий»

- [ ] Новый агент добавляется в комнату → smoke burst виден (9 puff-частиц)
- [ ] Smoke burst содержит символ `✦` или **не содержит никакого текста** (не `펑` — корейский символ)
- [ ] Агент появляется с анимацией: fade-in opacity 0→1 + scale 0.6→1.0 (~18 кадров)
- [ ] Агент удаляется из комнаты → fade-out opacity 1→0 + scale 1.0→0.8 (~12 кадров)
- [ ] Idle-состояние агента (working): лёгкое покачивание Y ±2px в цикле
- [ ] Offline-агент: серая иконка, нет покачивания, opacity снижена (~0.5)
- [ ] Sub-clone добавляется → firework burst через 3-4 секунды (≤ 210 кадров × 16ms ≈ 3.4с)

### 3.2 Режим B — «Корпоративный»

- [ ] Spawn: smoke burst с 5 puff-частицами (меньше, чем в игровом режиме)
- [ ] Burst text **отсутствует** (в корпоративном режиме убирается совсем)
- [ ] flash alpha при spawn ≤ 0.30 (не 0.52)
- [ ] Sub-clone firework burst раз в 6-7 секунд (≤ 420 кадров × 16ms ≈ 6.7с — реже, чем в игровом)
- [ ] Sub-clone анимация волны: медленнее (WAVE_SPEED ~0.025 vs 0.04), меньше амплитуда

---

## 4. Делегирование задач (Delivery Arc Animation)

### 4.1 Режим A — «Игровой и яркий»

- [ ] При делегировании задачи: конверт (envelope) вылетает из зоны CEO по дуге к целевой комнате
- [ ] Дуга высокая (крутая траектория) — визуально выражена
- [ ] Скорость доставки ~0.014 (быстрее корпоративного)
- [ ] Scale конверта: 1.1
- [ ] При прибытии: envelope bounce (scale 1.2→1.0, ~6 кадров)
- [ ] После arrival: smoke burst в целевом слоте агента
- [ ] После delivery: sub-clone добавляется с firework burst через ~3-4 секунды

### 4.2 Режим B — «Корпоративный»

- [ ] Дуга умеренная (меньше крутизна), плавная
- [ ] Скорость доставки ~0.010 (медленнее игрового)
- [ ] Scale конверта: 0.9 (меньше)
- [ ] **Нет** bounce при arrival (scale не скачет)
- [ ] Smoke burst после arrival: 4 puff-частицы

## 5. CEO Movement

### 5.1 Режим A — «Игровой и яркий»

- [ ] CEO движется между комнатами (видно визуальное перемещение спрайта)
- [ ] При входе CEO в комнату: brief white flash (~8 кадров)
- [ ] Частицы пыли при шагах CEO видны

### 5.2 Режим B — «Корпоративный»

- [ ] CEO движется медленнее (speed ~5 vs 7)
- [ ] **Нет** white flash при room-enter
- [ ] **Нет** частиц пыли при шагах

## 6. Meeting Badge Animations

- [ ] Статус совещания «kickoff»: badge пульсирует (scale 1.0→1.05→1.0, период ~2 секунды), цвет `#f59e0b`
- [ ] Статус «reviewing»: shimmer alpha 0.9→1.0, период ~1.5 секунды, цвет `#60a5fa`
- [ ] Статус «approved»: однократный bounce (scale 1.0→1.15→1.0, ~24 кадра), цвет `#34d399`
- [ ] Статус «hold»: flash alpha 1.0→0.6→1.0, период ~1 секунда, цвет `#f97316`

## 7. Интерактивность элементов

- [ ] Клик по агенту на канвасе: ripple-волна от точки клика (radius 0→24px, fade, ~20 кадров)
- [ ] Агент «подпрыгивает» при клике: Y -4px → 0 (spring easing, ~12 кадров)
- [ ] После клика по агенту: открывается AgentDetail-панель (React-transition справа)
- [ ] VirtualPadOverlay (мобильный): кнопки имеют press-feedback (scale 0.96 при нажатии)

## 9. Языковой контент — критичная проверка

- [ ] В smoke burst **отсутствует** текст `펑` (корейский символ)
- [ ] Весь текст в UI на русском или английском (в зависимости от выбранной локали)
- [ ] Никаких корейских символов нигде в анимациях или оверлеях

## 10. API-эндпоинты состояния анимаций комнат

### 10.1 GET /api/room-settings (список всех комнат)

- [ ] Возвращает HTTP 200
- [ ] Тело ответа: `{ ok: true, rooms: [...] }`
- [ ] Каждый элемент массива содержит: `room_id`, `settings`, `updated_by`, `updated_at`, `created_at`

### 10.2 GET /api/room-settings/:roomId (настройки конкретной комнаты)

- [ ] Возвращает HTTP 200 для существующей комнаты
- [ ] Тело ответа: `{ ok: true, room_id, settings, updated_by, updated_at, created_at }`
- [ ] Для несуществующей комнаты: HTTP 200 с `settings: {}`
- [ ] Для невалидного `roomId` (спецсимволы, слишком длинный): HTTP 400

### 10.3 PUT /api/room-settings/:roomId (обновление настроек — только admin)

- [ ] С заголовком `x-actor-role: admin` → HTTP 200, `{ ok: true, room_id }`
- [ ] С заголовком `x-actor-role: agent` → HTTP 403 `{ ok: false, error: "forbidden" }`
- [ ] Без заголовка (отсутствует) → по умолчанию admin → HTTP 200
- [ ] Тело запроса без поля `settings` → HTTP 400 `{ error: "settings_required" }`
- [ ] Поле `settings` не является объектом (например, строка или массив) → HTTP 400
- [ ] После успешного PUT: данные сохраняются и видны при GET

### 10.4 GET /api/room-settings-audit (журнал изменений — только admin)

- [ ] С заголовком `x-actor-role: admin` → HTTP 200, `{ ok: true, entries: [...] }`
- [ ] С заголовком `x-actor-role: agent` → HTTP 403
- [ ] После PUT admin-ом: в entries появляется новая запись с `actor_role: "admin"`
- [ ] Каждая запись содержит: `id`, `room_id`, `actor_role`, `change`, `prev`, `created_at`
- [ ] Параметр `?room_id=ceo_zone` фильтрует только записи для этой комнаты
- [ ] Параметр `?limit=5` ограничивает выдачу до 5 записей

## 11. WebSocket-события синхронизации анимаций

- [ ] При спавне агента (через UI или API) — в WebSocket-стриме приходит событие агента
- [ ] При делегировании задачи — delivery arc начинает анимацию **не позднее 500ms** после события
- [ ] При разрыве WS-соединения и реконнекте — анимации не «зависают» в промежуточном состоянии
- [ ] В консоли браузера нет ошибок WebSocket при нормальной работе

---

## 12. Производительность

- [ ] Офисный вид открывается без замораживания UI (< 1 секунды до интерактивности)
- [ ] FPS в офисном виде при наличии 5+ агентов — не ниже 30 fps (медиана)
- [ ] Переключение темы (light ↔ dark) не вызывает заметных фризов (< 50ms лага)
- [ ] В консоли браузера нет JS-ошибок при нормальной работе
